Role of AI in the Compliance Stack

08
AI is a Component, Not the Product

🧠 L2 Incident Planner

Claude API analyzes incident metadata and selects from 88 pre-approved runbooks. It does not generate arbitrary commands. Think of it as an intelligent router, not an autonomous agent.

  • Input: incident type, severity, system context (scrubbed)
  • Output: runbook_id + confidence score
  • Guardrail: ≥0.6 confidence required for auto-execution
  • Below threshold: escalates to human (L3)

↻ Data Flywheel

When L2 patterns succeed 5+ times at 90%+ rate, they are promoted to L1 deterministic rules. The system gets smarter over time while reducing AI dependency.

🛡 PHI Safeguards

12-pattern scrubber runs before any data reaches Claude API. Even if scrubbing fails, the input is system metadata — check types, severity levels, hostnames. Not clinical data.

Key Differentiators

Runbook-Constrained

The LLM selects from a fixed set of 88 runbooks. It cannot generate shell commands, modify configurations, or take actions outside the approved set. This is a selection problem, not a generation problem.

Self-Reducing AI Dependency

The Data Flywheel continuously promotes L2 patterns to L1. As the system learns, AI calls decrease. The end state is a deterministic system that rarely needs LLM inference.

Cost Structure

L1: $0/incident • L2: ~$0.001/incident • At 1,000 incidents/day, AI cost is <$6/month total across all tenants.
