# compliance_packets_stalled

**Severity:** sev1
**Display name:** HIPAA monthly attestation packet missing

## What this means (plain English)

`compliance_packets` (mig 141) is the table that holds **monthly
HIPAA compliance attestations** auto-generated by
`_compliance_packet_loop`. HIPAA §164.316(b)(2)(i) requires **6-year
retention** of these packets — they are auditor-visible artifacts.

This invariant fires when:
1. A site emitted `compliance_bundles` in the prior completed month
   (proving it was operationally active that month), AND
2. NO `compliance_packets` row exists for that site + month + year
   with `framework='hipaa'`, AND
3. We are >24h past the start of the current month (the auto-gen
   loop runs hourly; 24h is generous grace).

A fire means the auto-gen path silently dropped a month — that's a
real, durable HIPAA retention gap until backfilled.

## Why this matters (architectural)

- `_compliance_packet_loop` runs hourly per CLAUDE.md. Each pass
  catches up the most recent N=3 unfinished months. If the loop is
  wedged or every iteration errors, months accumulate.
- Pre-Session 203 there was a `now.day == 1 AND now.hour == 2`
  one-hour-window check that lost a packet on every container restart
  during that window. The current code does NOT have that bug, but a
  regression that re-introduces a tight schedule would silently break
  attestation continuity.
- Block 3 (commit 7e71fc35) upgraded the per-packet failure log from
  `logger.warning` to `logger.error` so the log shipper alerts. That
  closes the upstream observability gap; this invariant closes the
  outcome-layer gap.

## Root cause categories

- **Auto-gen path failed silently for one specific site.** Bug in
  the markdown_content rendering for that site's data shape; bug
  in the controls_summary aggregation. Check
  `compliance_packet_autogen_failed` ERROR log lines.
- **Auto-gen loop wedged for ALL sites.** `_compliance_packet_loop`
  hangs on an asyncpg call. `bg_loop_silent` should ALSO fire on
  `compliance_packets` — cross-correlate.
- **Schema drift.** A migration removed a column the loop depends on
  (e.g. `compliance_bundles.compliance_score` if removed); the loop
  errors on every iteration but `_supervised` keeps it running.
- **Operator manually deleted historical packets.** Migration 151
  added a DELETE-blocking trigger on `compliance_packets` should
  prevent this; if rows ARE missing, the trigger may have been
  bypassed (DBA-only operation, audit-log will show).

## Immediate action

1. **Confirm the gap is real, not a fixture/test artifact:**
   ```sql
   SELECT site_id, year, month, generated_at
     FROM compliance_packets
    WHERE site_id = '<site>' AND framework = 'hipaa'
    ORDER BY year DESC, month DESC LIMIT 12;
   ```

2. **Check the auto-gen log for that site+month:**
   ```
   ssh root@<vps> "docker compose -f /opt/mcp-server/docker-compose.yml \
     logs mcp-server --since 7d 2>/dev/null | \
     grep -E 'compliance_packet_autogen_failed.*<site_id>'"
   ```

3. **Check if `compliance_packets` loop is healthy** (sibling
   invariant `bg_loop_silent` would fire if not).

4. **Manual backfill** via the admin endpoint (preferred path —
   replays the same generator that the loop would have used):
   ```
   curl -X POST -H "Authorization: Bearer $ADMIN_TOKEN" \
     "https://api.osiriscare.net/api/admin/compliance/packets/backfill" \
     -d '{"site_id": "<site>", "year": <year>, "month": <month>}'
   ```

5. **Per CLAUDE.md non-operator partner posture:** substrate
   exposes the gap; the operator (MSP / partner) decides per-BAA
   disclosure. For routine auto-gen failures that backfill cleanly,
   no clinic notification needed. For systematic multi-month gaps
   across many sites, partner round-table consults whether the
   incident materially affects the customer's audit posture.

## Verification

- Panel: invariant row resolves on the next 60s tick after the
  packet INSERTs successfully.
- CLI: re-run query (1) — the missing year+month row should now
  exist.

## Escalation

Sev1 — operator action expected within the workday. **NOT** paging-
class because the gap is durable; no in-flight evidence is at risk.

If the same site fires this for 3+ months in a row, the auto-gen
path is broken for that site specifically — escalate to engineering
for a root-cause fix beyond manual backfill.
