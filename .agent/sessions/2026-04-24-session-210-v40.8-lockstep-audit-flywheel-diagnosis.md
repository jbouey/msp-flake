# Session 210 — v40.8 ISO, lockstep audit, flywheel diagnosis

**Date:** 2026-04-24
**Focus:** Close v40.7 canary findings, run first lockstep audit across two
highest-risk contracts, diagnose the "no promotions in a month" signal.

## What triggered the session

v40.7 canary on .242 (1D:0F:E5) reached the `msp-breakglass-submit` stage
for the first time and immediately failed with `submit: site_id/api_key/
appliance_id missing — retry in 5m`. This bug had been latent since v40.0
but was masked in every prior ISO by earlier boot failures (Phase 0
deadlock, classpath, em-dash). First canary to reach this stage exposed it.

## Closed this session

  includes `appliance_id`. Fresh boxes get a complete config.yaml.

  fallback (mirrors `msp-journal-upload`) so legacy v40.0-v40.7 boxes

  into `.githooks/pre-push`.

  working breakglass submit with NO new ISO cut required. Backend-only
  fix propagated to every reinstalled box automatically. ADR
  `2026-04-24-source-of-truth-hygiene.md` thesis validated in production.

### 5. Flywheel diagnosis — "no promotions in 30 days"

Walked the pipeline in 6 stages (coach-style audit template). Root cause
was NOT a bug — **L2 was intentionally killed by commit `b0a2b443` on
2026-04-12 02:31 UTC** with a `L2_ENABLED=false` kill switch. Original
justification (0.9% success rate, zero promoted_rule deployments, unbounded
API spend) has since been fixed by Session 206's Flywheel Spine +
three cost gates. User decision still pending on whether to flip
`L2_ENABLED=true` with a 72h watch.

Subsidiary findings (all fixed):
- 3 stuck `promotion_eligible=true` rows in `aggregated_pattern_stats`
  pointing at dead/historical site_ids. Cleanup: 9 rows deleted, 2
  un-stuck.
- `sigauth_auto_promotion` tick loop crashed every iteration
  (`'str' object has no attribute 'days'` — passing string to
  `$1::interval` asyncpg binder). Commit `108127f5` replaces with
  `timedelta(hours=AUTO_PROMOTE_WINDOW_HOURS)`.
- 2 phantom `L2-%` rows in `platform_pattern_stats` remain despite
  migration 237's DELETE; marker for future re-investigation
  (non-urgent — the Step-4 SELECT filters them out now via threshold).

## Key principles validated today

1. **Source-of-truth hygiene (ADR 2026-04-24)** — backend-only fix
   healed every reinstalled appliance without a new ISO. Fix at the
   producing layer, not the consuming layer.
2. **Lockstep audit as a reusable tool** — 400-line prompt, 5 real
   drift findings in under 10 minutes, 1 CRITICAL + 2 HIGH. Worth
   re-running against evidence_bundle + fleet_order contracts next.
3. **Canary discipline** — v40.7 reflash seemed redundant ("we already
   fixed v40.6's bug") but reaching a new boot stage exposed a
   pre-existing bug that had been hidden by earlier failures. Each
   green canary unlocks visibility into the next latent class.
